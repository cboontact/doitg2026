# 📘 คู่มือการใช้งานระบบกิจการนักเรียนดอยหัวเสือ

เอกสารฉบับนี้เขียนให้ใช้งานได้จริงทั้งตอนอบรมทีมและตอนเปิดระบบใช้ในโรงเรียน โดยเน้นให้อ่านง่าย เห็นภาพรวมไว และมีโฟลการทำงานของแต่ละส่วนชัดเจน

## 🌐 1. ภาพรวมของระบบ

ระบบกิจการนักเรียนดอยหัวเสือเป็นเว็บแอปสำหรับงานดูแลนักเรียนในแต่ละวัน โดยรวมงานสำคัญไว้ในระบบเดียว ได้แก่

- ✅ เช็คชื่อหน้าเสาธง
- ⏰ บันทึกมาสาย
- 🚶 ขอออกนอกบริเวณและสแกนออก-กลับ
- ⭐ จัดการคะแนนความประพฤติ
- 🧠 ประเมิน SDQ
- 👩‍🏫 จัดการครูที่ปรึกษา
- 👤 จัดการผู้ใช้งาน
- 🧑‍🎓 พอร์ทัลนักเรียน
- 📝 ใบลาของนักเรียน
- 🎓 เลื่อนชั้นนักเรียน
- ⚙️ ตั้งค่าปีการศึกษาและวันหยุด

ระบบนี้ทำงานบน Cloudflare เป็นหลัก โดยใช้ Cloudflare Workers สำหรับ runtime และ Cloudflare D1 สำหรับฐานข้อมูล

## 🎯 2. เป้าหมายของระบบ

ระบบนี้ถูกออกแบบมาเพื่อแก้ปัญหา 4 เรื่องหลัก

1. ลดงานกระดาษและการจดซ้ำ
2. ให้ครูและเจ้าหน้าที่เห็นข้อมูลนักเรียนแบบทันที
3. ให้ข้อมูลการมาเรียน มาสาย ลา และพฤติกรรมเชื่อมกัน
4. เปิดทางให้นักเรียนตรวจสอบข้อมูลของตนเองได้

## 👥 3. บทบาทผู้ใช้งาน

ระบบรองรับ 4 บทบาทหลัก

### 👑 3.1 ผู้ดูแลระบบ `admin`

สิทธิ์หลัก:

- จัดการผู้ใช้งาน
- จัดการครูที่ปรึกษา
- จัดการนักเรียน
- ตั้งค่าระบบ
- เลื่อนชั้น
- ดูและแก้ไขข้อมูลได้ทุกส่วน

### 👩‍🏫 3.2 ครู `teacher`

สิทธิ์หลัก:

- เช็คชื่อหน้าเสาธง
- ดูข้อมูลนักเรียนในความดูแล
- ลงทะเบียนออกนอกบริเวณ
- ดูและแก้ไขคะแนนความประพฤติในขอบเขตที่ได้รับสิทธิ์
- เห็นใบลาของนักเรียนในหน้าเช็คชื่อ

หมายเหตุ:

- ครูที่ปรึกษาจะเห็นห้องของตนเองขึ้นก่อนในหลายหน้า
- ถ้าครูมีสิทธิ์ไม่ครบทุกห้อง ระบบจะจำกัดข้อมูลตามสิทธิ์

### 🧰 3.3 เจ้าหน้าที่ `staff`

สิทธิ์หลัก:

- บันทึกมาสาย
- เช็คชื่อหน้าเสาธง
- จัดการออกนอกบริเวณ
- ใช้งานจุดสแกน
- ดูข้อมูลนักเรียน
- ดูข้อมูลครูที่ปรึกษา
- จัดการคะแนนความประพฤติ

### 🧑‍🎓 3.4 นักเรียน `student`

สิทธิ์หลัก:

- ดูคะแนนความประพฤติของตัวเอง
- ดูประวัติการถูกเพิ่มและหักคะแนน
- ยื่นใบลา
- ดูสถานะใบลาของตัวเอง

## 🗺️ 4. หน้าใช้งานหลักของระบบ

### 🏠 4.1 หน้าหลักของระบบ

- `/login` หน้าเข้าสู่ระบบ
- `/dashboard` หน้าหลักสำหรับผู้ดูแล ครู และเจ้าหน้าที่
- `/student` หน้าหลักของนักเรียน

### 🛠️ 4.2 หน้าภายในแดชบอร์ด

- `/dashboard/attendance` เช็คชื่อหน้าเสาธง
- `/dashboard/late-checkin` บันทึกมาสาย
- `/dashboard/leave-requests` ขอออกนอกบริเวณ
- `/dashboard/leave-checkpoint` จุดสแกนออก-กลับ
- `/dashboard/behavior` คะแนนความประพฤติ
- `/dashboard/sdq` ประเมิน SDQ
- `/dashboard/students` รายชื่อนักเรียน
- `/dashboard/teachers` ครูที่ปรึกษา
- `/dashboard/teacher-registrations` ลงทะเบียนครู
- `/dashboard/users` ผู้ใช้งาน
- `/dashboard/promotions` เลื่อนชั้น
- `/dashboard/settings` ตั้งค่าระบบ

### 🌍 4.3 หน้าสาธารณะ

- `/late-checkin` จุดบันทึกมาสาย
- `/leave-checkpoint` จุดสแกนออก-กลับ
- `/register/teacher` หน้าสมัครครู
- `/sdq/parent` หน้าประเมิน SDQ สำหรับผู้ปกครอง

หน้าสาธารณะเหมาะสำหรับเปิดค้างไว้ที่จุดใช้งานจริง เช่น จุดสแกนหน้าประตูหรือจุดออกนอกบริเวณ

## 🔐 5. การเข้าสู่ระบบ

### 👨‍💼 5.1 ผู้ดูแล ครู และเจ้าหน้าที่

1. เข้า `/login`
2. กรอกชื่อผู้ใช้
3. กรอกรหัสผ่าน
4. กด `เข้าสู่ระบบ`
5. ระบบจะพาไป `/dashboard`

### 🧑‍🎓 5.2 นักเรียน

1. เข้า `/login`
2. กรอก `ชื่อผู้ใช้ = เลขประจำตัวนักเรียน`
3. กรอก `รหัสผ่าน = เลขประจำตัวนักเรียน`
4. กด `เข้าสู่ระบบ`
5. ระบบจะพาไป `/student`

หมายเหตุ:

- นักเรียน login จากข้อมูลใน `students` โดยตรง ไม่ต้องมีบัญชีแยกในเมนู `ผู้ใช้งาน`
- ถ้าไม่มีการใช้งานเกิน 30 นาที ระบบจะให้ออกจากระบบและ login ใหม่

## 🧱 6. โครงสร้างสถาปัตยกรรมระบบ

ส่วนนี้อธิบายว่า “ระบบแบ่งชั้นการทำงานอย่างไร”

### 🏗️ 6.1 โครงสร้างระดับสูง

```text
+------------------+     +--------------------+     +----------------------+
|   ผู้ใช้งาน      | --> |   หน้าเว็บ Next.js | --> |   API ภายในระบบ      |
+------------------+     +--------------------+     |   (/api/*)           |
                                                     +----------------------+
                                                                |
                                                                v
                                                     +----------------------+
                                                     | business logic       |
                                                     | ใน src/lib           |
                                                     +----------------------+
                                                                |
                                                                v
                                                     +----------------------+
                                                     |  Cloudflare D1       |
                                                     |  ฐานข้อมูลหลัก       |
                                                     +----------------------+
```

### 🔩 6.2 ส่วนประกอบหลัก

- `Frontend`
  ใช้ Next.js App Router แสดงหน้า dashboard, หน้า login, หน้า public และพอร์ทัลนักเรียน

- `Backend API`
  ใช้ Route Handlers ใน `src/app/api` เป็น API ภายใน เช่น auth, attendance, behavior, leave requests และ students

- `Authentication`
  ใช้ cookie session และ middleware สำหรับควบคุมสิทธิ์การเข้าถึงหน้าและ API

- `Business Logic`
  เก็บใน `src/lib` เช่น กฎการหักคะแนน การตรวจสิทธิ์ การคำนวณสถานะ และการเชื่อมข้อมูลหลายตาราง

- `Database`
  ใช้ Cloudflare D1 เป็นฐานข้อมูลหลักของระบบ

### 🧩 6.3 โมดูลหลักของระบบ

- `โมดูลยืนยันตัวตน`
  login, logout, เปลี่ยนรหัสผ่าน, session timeout

- `โมดูลข้อมูลหลัก`
  users, teachers, students, academic years, holidays

- `โมดูลงานประจำวัน`
  attendance, late check-in, leave checkpoint

- `โมดูลพฤติกรรม`
  behavior score, behavior logs, score rules

- `โมดูล SDQ`
  sdq periods, teacher assessment, student self-assessment, parent public assessment, summary dashboard

- `โมดูลนักเรียน`
  student portal, student leave requests, ดูคะแนนของตัวเอง

- `โมดูลลงทะเบียนครู`
  teacher directory, teacher registration requests, approve / reset, import CSV

### 🗄️ 6.4 ตารางฐานข้อมูลหลัก

- `users` บัญชีผู้ใช้ของระบบ
- `teachers` ข้อมูลครูที่ปรึกษาและสิทธิ์ที่เกี่ยวข้อง
- `teacher_directory` รายชื่อครูที่เปิดให้ลงทะเบียน
- `teacher_registration_requests` คำขอลงทะเบียนครูที่รอแอดมินอนุมัติ
- `students` ข้อมูลนักเรียนหลัก
- `attendance` การเช็คชื่อประจำวัน
- `late_checkins` รายการมาสาย
- `leave_requests` คำขอออกนอกบริเวณ
- `behavior_logs` ประวัติการเพิ่มและหักคะแนน
- `score_rules` กฎคะแนน
- `sdq_periods` รอบประเมิน SDQ
- `sdq_assessments` หัวรายการประเมิน SDQ
- `sdq_answers` คำตอบรายข้อของ SDQ
- `sdq_scores` คะแนนสรุป SDQ
- `academic_years` ปีการศึกษา
- `special_holidays` วันหยุดพิเศษ
- `student_leave_requests` ใบลาของนักเรียน

## 🔄 7. โฟลการทำงานของระบบ

ส่วนนี้อธิบาย “ใครกดอะไร แล้วข้อมูลไหลไปไหน”

### 🔐 7.1 โฟลการ login

```text
+------------------------+
| ผู้ใช้กรอกข้อมูล       |
+------------------------+
            |
            v
+------------------------+
| หน้า /login            |
+------------------------+
            |
            v
+------------------------+
| /api/auth/login        |
+------------------------+
      |             |
      v             v
+-----------+        +-------------+
|  users    |        |  students   |
+-----------+        +-------------+
      |                     |
      |                     |
      |             login ด้วยเลขประจำตัว
      |                     |
      +----------+----------+
                 |
                 v
      +----------------------+
      | สร้าง session cookie |
      +----------------------+
                 |
                 v
      +----------------------+
      | ตรวจบทบาทผู้ใช้      |
      +----------------------+
           |             |
           v             v
    +-------------+   +-----------+

    +-------------+   +-----------+
```

ขั้นตอน:

1. ผู้ใช้กรอกชื่อผู้ใช้และรหัสผ่าน
2. หน้า login ส่งข้อมูลไป `/api/auth/login`
3. ถ้าเป็นนักเรียน ระบบตรวจใน `students` โดยตรง
4. ถ้าเป็นบุคลากร ระบบตรวจบัญชีใน `users`
5. ระบบสร้าง token และ cookie
6. พาไปหน้าตามบทบาทของผู้ใช้

3. ระบบตรวจว่าข้อมูลเวลาใช้ได้หรือไม่
4. ระบบตรวจว่านักเรียนมีคำขอค้างอยู่หรือไม่
5. ถ้าเป็นครูที่ปรึกษาส่งคำขอ ระบบจะบันทึกเป็น `รอยืนยัน`
6. เจ้าหน้าที่หรือผู้ดูแลระบบกด `ยืนยันรายการ` แล้วคำขอจะเปลี่ยนเป็น `รอออก`
7. ถ้าเจ้าหน้าที่หรือผู้ดูแลระบบเป็นคนส่งคำขอเอง ระบบจะพร้อมสแกนทันที
8. เมื่อนักเรียนไปถึงจุดสแกน ระบบจะบันทึกเวลาออกจริง
9. ตอนนักเรียนกลับ ระบบจะบันทึกเวลากลับจริง
10. ถ้ากลับช้าหรือไม่กลับ ระบบอาจหักคะแนนและบันทึกลง `behavior_logs`

## 👑 8. คู่มือสำหรับผู้ดูแลระบบ

### 🚀 8.1 สิ่งที่ควรตรวจหลัง login

1. วันที่ระบบถูกต้อง
2. ปีการศึกษาที่ active ถูกต้อง
3. การ์ดสรุปข้อมูลไม่ผิดปกติ
4. ไม่มีห้องหรือรายชื่อนักเรียนหาย

### 📊 8.1.1 หน้าหลัก dashboard

ใช้สำหรับ:

- ดูสรุปข้อมูลภาพรวมของวัน
- ตรวจจำนวนขาด มาสาย ลา หรือเหตุผิดปกติ
- ใช้เป็นจุดเริ่มต้นก่อนเข้าเมนูงานประจำวัน

ลักษณะการใช้งาน:

- หลัง login ระบบจะพาเข้า `/dashboard`
- ควรตรวจการ์ดสรุปและรายการล่าสุดก่อนทุกเช้า
- ถ้าตัวเลขผิดปกติ ให้ไล่ตรวจต่อที่ `เช็คชื่อ`, `มาสาย`, `ออกนอกบริเวณ`, หรือ `คะแนนความประพฤติ`

### 👤 8.2 เมนูผู้ใช้งาน

ใช้สำหรับ:

- เพิ่มผู้ใช้ใหม่
- แก้ไขข้อมูลผู้ใช้
- เปลี่ยนบทบาท
- รีเซ็ตรหัสผ่าน
- เปิดหรือปิดบัญชี

หมายเหตุ:

- เมนูนี้ใช้สำหรับ `admin`, `teacher`, `staff`
- ไม่ใช้สร้างบัญชีนักเรียน
- นักเรียนใช้เลขประจำตัวเป็นชื่อผู้ใช้และรหัสผ่านจากเมนู `รายชื่อนักเรียน`

ลำดับการใช้งาน:

1. กด `เพิ่มผู้ใช้`
2. กรอกชื่อผู้ใช้ รหัสผ่าน ชื่อ และนามสกุล
3. เลือกบทบาท เช่น `admin`, `teacher`, `staff`
4. ถ้าต้องการบังคับเปลี่ยนรหัสผ่าน ให้ตั้งค่าตอนสร้างหรือแก้ไข
5. บันทึก แล้วทดสอบ login ด้วยบัญชีนั้น

### 👩‍🏫 8.3 เมนูลงทะเบียนครู

- ดูแท็บ `คำขอลงทะเบียน`
- ดูแท็บ `เพิ่มรายชื่อครู`
- เพิ่มรายชื่อทีละคน
- import รายชื่อด้วย CSV
- ดาวน์โหลดเท็มเพลต CSV
- อนุมัติ, ไม่อนุมัติ, หรือยกเลิกคำขอที่ยังไม่อนุมัติ
- ลบรายชื่อครูที่ยังไม่ถูกอนุมัติ
- ลบคำขอลงทะเบียนที่ยังไม่ถูกอนุมัติ

ลักษณะการใช้งาน:

- แท็บ `คำขอลงทะเบียน` ใช้ดูคำขอของครูที่ส่งเข้ามา
- แท็บ `เพิ่มรายชื่อครู` ใช้จัดการรายชื่อครูที่เปิดสิทธิ์ให้สมัคร
- ปุ่ม `ดาวน์โหลดเท็มเพลต`, `อัปโหลด CSV`, และ `เพิ่มรายชื่อครู` จะอยู่ในแท็บ `เพิ่มรายชื่อครู`
- ถ้าครูคนใด `อนุมัติแล้ว` รายชื่อนั้นจะแสดงเพียงสถานะ `อนุมัติแล้ว` และจะไม่แก้ไขจากเมนูนี้
- ถ้ารายชื่อครูยังไม่ถูกอนุมัติ สามารถลบออกจากรายการได้
- ถ้าคำขอลงทะเบียนยังไม่ถูกอนุมัติ สามารถลบคำขอได้
- หากต้องการเปลี่ยนข้อมูลหลังอนุมัติ ให้ไปแก้ที่เมนู `ผู้ใช้งาน` หรือ `ครูที่ปรึกษา`

### 👩‍🏫 8.4 เมนูครูที่ปรึกษา

ใช้สำหรับ:

- ผูกครูกับห้อง
- ระบุหัวหน้าระดับ
- ระบุขอบเขตการดูแล
- ให้เจ้าหน้าที่ดูข้อมูลได้แบบอ่านอย่างเดียว

### 🧑‍🎓 8.5 เมนูรายชื่อนักเรียน

ใช้สำหรับ:

- เพิ่มและแก้ไขนักเรียน
- import ข้อมูล
- ตรวจ RFID
- ตรวจคะแนนตั้งต้น
- ใช้เป็นแหล่งข้อมูล login ของนักเรียนโดยตรง
- เลือก `ระดับชั้น` และ `ห้อง` แล้วกดค้นหา
- ดูผลลัพธ์แบบแบ่งหน้า และเลือกจำนวนรายการต่อหน้า `30 / 40 / 50 / 100`

ลำดับการใช้งาน:

1. เลือก `ระดับชั้น` และ `ห้อง` หรือพิมพ์ค้นหาชื่อ/เลขประจำตัว
2. กด `ค้นหา`
3. ตรวจรายการในตาราง
4. เพิ่มหรือแก้ไขข้อมูลนักเรียนตามต้องการ
5. ถ้าจะนำเข้าจำนวนมาก ให้ใช้ไฟล์ import ตามเท็มเพลต

### 📏 8.5.1 เมนูเกณฑ์คะแนน

ใช้สำหรับ:

- สร้างกฎคะแนนมาตรฐานของโรงเรียน
- แยกกฎ `เพิ่ม` และ `หัก`
- เปิดหรือปิดกฎที่ไม่ใช้แล้ว

ลำดับการใช้งาน:

1. เปิดเมนู `เกณฑ์คะแนน`
2. กดเพิ่มกฎใหม่ หรือเลือกแก้ไขกฎเดิม
3. กำหนดชื่อกฎ ประเภท จำนวนคะแนน และหมวดหมู่
4. บันทึก
5. ทดสอบว่าเมนู `คะแนนความประพฤติ` เรียกกฎนั้นได้ถูกต้อง

### 🧠 8.6 เมนูประเมิน SDQ

ใช้สำหรับ:

- สร้างรอบประเมิน SDQ
- ใช้งานผ่านแท็บ `ภาพรวมการประเมิน`
- ใช้งานผ่านแท็บ `จัดรอบประเมิน`
- เปิดหรือสลับรอบประเมินที่ใช้งาน
- เปิดหรือปิดช่องทาง `ครู / นักเรียน / ผู้ปกครอง`
- แก้ไขรอบประเมินที่สร้างแล้ว
- ลบรอบประเมินที่ไม่ต้องการ
- ดูว่านักเรียนแต่ละคนถูกประเมินครบกี่ฝั่งแล้ว
- ดูคะแนนสรุป SDQ ของแต่ละฝั่ง

ลักษณะการใช้งาน:

- แท็บ `ภาพรวมการประเมิน` ใช้ดูสถิติ ค้นหารายชื่อ และเปิดประเมินรายคน
- แท็บ `จัดรอบประเมิน` ใช้สร้าง แก้ไข ลบ และเปิดใช้งานรอบประเมิน
- แอดมินจะเห็นส่วน `จัดการรอบประเมิน`
- ฟอร์ม `สร้างรอบประเมิน` และ `แก้ไขรอบประเมิน` เปิดเป็น modal ไม่ค้างบนหน้า
- หน้ารายชื่อนักเรียนในเมนูนี้จะยังไม่แสดงทันที จนกว่าจะเลือกตัวกรองหรือพิมพ์ค้นหาแล้วกด `ค้นหา`
- ถ้าเป็น `ครู` ระบบจะใส่ `ชั้น/ห้องที่ปรึกษา` ของตนเองเป็นค่าเริ่มต้นให้ก่อน
- ครูจะเห็นเฉพาะรายการนักเรียนที่ตนมีสิทธิ์ประเมิน
- ถ้ายังไม่มีรอบประเมิน หรือยังไม่เปิดให้ครูประเมิน ปุ่ม `ประเมิน` จะถูกล็อกไว้
- ผู้ปกครองใช้ลิงก์ `/sdq/parent` แยกจาก dashboard
- การ์ดสรุปด้านบนจะแสดงทั้ง `จำนวนคน` และ `เปอร์เซ็นต์`
- ในตารางของแต่ละฝั่งจะมี 2 badge คือ `สถานะการประเมิน` และ `ผลแปล`
- ผลแปลจะแสดงเป็น badge เช่น `ปกติ`, `เสี่ยง`, `มีปัญหา`

เกณฑ์การแปลผล:

- `0-13` = `ปกติ`
- `14-16` = `เสี่ยง`
- `17 ขึ้นไป` = `มีปัญหา`
- ระบบใช้คะแนนรวมจาก `อารมณ์`, `ความประพฤติ`, `สมาธิ/อยู่ไม่นิ่ง`, และ `เพื่อนสัมพันธ์`
- คะแนน `สัมพันธภาพทางสังคม` แสดงแยก และไม่รวมในคะแนนปัญหา

### ⚙️ 8.7 เมนูตั้งค่าระบบ

ใช้สำหรับ:

- ปีการศึกษา
- วันหยุดพิเศษ
- ช่วงเปิดภาคเรียน

ลำดับการใช้งาน:

1. สร้างหรือเลือก `ปีการศึกษา`
2. กำหนดวันเปิด-ปิดภาคเรียน
3. ตั้งให้ปีที่ถูกต้องเป็น `active`
4. เพิ่มวันหยุดพิเศษ
5. ตรวจว่าการตั้งค่าถูกใช้ในเมนูอื่นแล้ว

### 🎓 8.8 เมนูเลื่อนชั้น

ใช้สำหรับ:

- ย้ายนักเรียนไปปีถัดไป
- จบชั้นสำหรับชั้นปลาย

ลำดับการใช้งาน:

1. สร้างปีการศึกษาปลายทางก่อน
2. เปิดเมนู `เลื่อนชั้นนักเรียน`
3. ตรวจจำนวนและรายละเอียดนักเรียน
4. ยืนยันการเลื่อนชั้น
5. ตรวจซ้ำที่เมนู `รายชื่อนักเรียน`

## 👩‍🏫 9. คู่มือสำหรับครู

### 📝 9.0 การสมัครใช้งานครู

1. รับลิงก์หน้าสมัครจากแอดมิน
2. เลือกชื่อครูของตนเอง

4. ส่งคำขอลงทะเบียน
5. รอแอดมินอนุมัติ
6. เมื่ออนุมัติแล้วจึง login เข้าระบบได้

### ✅ 9.1 เช็คชื่อหน้าเสาธง

ลำดับการใช้งาน:

1. เลือกห้อง
2. เช็กสถานะนักเรียน
3. ดูคำแนะนำจากใบลาถ้ามี
4. กดบันทึก

### 🚶 9.2 ขอออกนอกบริเวณ

ลำดับการใช้งาน:

1. ค้นหานักเรียน
2. กรอกข้อมูลเวลา

4. กดบันทึกคำขอ

### ⭐ 9.3 คะแนนความประพฤติ

ลำดับการใช้งาน:

1. เลือกชั้นและห้อง
2. ค้นหานักเรียน
3. ตรวจคะแนนเดิม
4. เพิ่มหรือหักคะแนน
5. บันทึก

### 🧠 9.4 ประเมิน SDQ ของครู

ลำดับการใช้งาน:

1. เข้าเมนู `ประเมิน SDQ`
2. ระบบจะเลือก `ชั้น/ห้องที่ปรึกษา` ของครูให้เป็นค่าเริ่มต้น
3. เลือกตัวกรองหรือพิมพ์ค้นหานักเรียน แล้วกด `ค้นหา`
4. กดปุ่ม `ประเมิน`
5. ตอบคำถามทั้ง 25 ข้อ โดยปุ่มตัวเลือกจะมีขนาดพอดีกับข้อความ แสดงสี และมีไอคอนเมื่อเลือกแล้ว
6. ถ้ายังไม่เปิดรอบประเมิน หรือยังไม่เปิดให้ครูประเมิน ปุ่ม `ประเมิน` จะกดไม่ได้
7. ระบบจะยังไม่บันทึกอัตโนมัติ ต้องกด `บันทึกการประเมิน`
8. หลังบันทึก ระบบจะคำนวณผลแปลทันที
9. ตรวจดูสถานะของ `ครู / นักเรียน / ผู้ปกครอง` ในตารางสรุป
10. ดู badge ผลแปลใต้คำว่า `ประเมินแล้ว` ของแต่ละฝั่งได้ทันที

- ใช้ค้นหารายชื่อนักเรียนเพื่อดูสถานะ แต่ไม่ใช้ส่งแบบประเมินแทนครู
- ดูเปอร์เซ็นต์จากการ์ดสรุป และดูผลแปลจาก badge ของแต่ละฝั่งได้

## 👨‍👩‍👧‍👦 12. คู่มือสำหรับผู้ปกครอง

### 🧠 12.1 ประเมิน SDQ ของผู้ปกครอง

ลำดับการใช้งาน:

1. เปิดลิงก์ `/sdq/parent`
2. กรอก `เลขประจำตัวนักเรียน`
3. ตรวจสอบชื่อและห้องของนักเรียนให้ถูกต้อง
4. ตอบคำถามทั้ง 25 ข้อตามพฤติกรรมของนักเรียน โดยปุ่มตัวเลือกจะแสดงสีและไอคอนเมื่อเลือกแล้ว

6. ระบบจะคำนวณคะแนนและบันทึกผลให้ทันที

ลักษณะการใช้งาน:

- ผู้ปกครองต้องกด `ค้นหา` ก่อนจึงจะเข้าสู่แบบประเมินได้
- ถ้านักเรียนคนนี้ถูกประเมินโดยผู้ปกครองแล้ว ระบบจะแจ้งว่า `ส่งแบบประเมิน SDQ แล้ว`
- หลังส่งแล้ว หน้าจอจะแสดงผลสรุปของผู้ปกครองแทนแบบฟอร์ม
- ถ้ายังไม่มีรอบประเมินหรือไม่พบเลขประจำตัว ระบบจะแจ้งข้อผิดพลาดบนหน้าเดียวกัน

## 🧪 13. ปัญหาที่พบบ่อย

### ❌ 13.1 login ไม่ได้

ให้ตรวจ:

1. ชื่อผู้ใช้ถูกหรือไม่
2. รหัสผ่านถูกหรือไม่
3. ถ้าเป็นนักเรียน ใช้เลขประจำตัวทั้งสองช่องหรือไม่
4. บัญชียัง active หรือไม่

### 👀 13.2 ไม่เห็นรายชื่อนักเรียน

ให้ตรวจ:

1. เลือกห้องถูกหรือไม่
2. สิทธิ์ครูถูกตั้งหรือยัง
3. นักเรียนยัง active หรือไม่

### 📍 13.3 สแกนออกไม่ได้

ให้ตรวจ:

1. มีคำขอออกนอกบริเวณก่อนหรือยัง
2. สแกนบัตรถูกคนหรือไม่
3. คำขอเดิมยังค้างหรือไม่

### 🧠 13.4 SDQ ประเมินไม่ได้หรือไม่ขึ้นแบบฟอร์ม

ให้ตรวจ:

1. มี `รอบประเมิน SDQ` ที่เปิดใช้งานอยู่หรือไม่
2. ช่องทางของผู้ประเมินนั้นถูกเปิดไว้หรือไม่ เช่น `ครู`, `นักเรียน`, `ผู้ปกครอง`
3. ถ้าเป็นครู ปุ่ม `ประเมิน` ถูกล็อกเพราะยังไม่เปิดให้ครูหรือไม่
4. ถ้าเป็นนักเรียนหรือผู้ปกครอง อาจส่งแบบประเมินไปแล้ว ระบบจึงแสดงผลสรุปแทนแบบฟอร์ม
5. ถ้าเป็นผู้ปกครอง ตรวจเลขประจำตัวนักเรียนว่าถูกต้องหรือไม่

## 💡 14. ข้อแนะนำก่อนใช้จริง

- ควรตั้งปีการศึกษาให้ถูกก่อนเปิดใช้งาน
- ควร import นักเรียนและตรวจห้องให้ครบ
- ควรผูกครูที่ปรึกษาให้ครบก่อนวันเปิดเทอม
- ควรทดสอบจุดสแกนทุกเช้า
- ควรให้นักเรียนทดลอง login ก่อนใช้จริง
- ควรสำรองข้อมูล D1 เป็นระยะ

## ✅ 15. สรุป

ถ้าจะเริ่มใช้ระบบนี้แบบเรียงลำดับที่ถูก ควรทำดังนี้

4. ผูกครูที่ปรึกษากับห้อง
5. ทดสอบ login ทุกบทบาท
